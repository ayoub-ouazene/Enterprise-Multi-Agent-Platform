# 17. Future Improvements

The following features are explicitly postponed, not forgotten.

## 17.1 Company Data Integrations

- read-only database views;
- company APIs;
- ERP connectors;
- HR-system connectors;
- finance-system connectors;
- hybrid uploaded and connected data;
- controlled write adapters or stored procedures.

## 17.2 Dedicated Enterprise Deployments

Large customers may later receive:

- dedicated database;
- dedicated vector index;
- private deployment;
- custom retention policies.

## 17.3 New Departments

Version 1 supports only five departments.

Future versions may add:

- Legal and Compliance;
- Sales;
- Marketing;
- Operations;
- Supply Chain.

Adding user-created departments requires a later plugin architecture and is not part of Version 1.

## 17.4 Advanced Workflow Infrastructure

Possible future additions:

- managed background queues;
- event bus;
- distributed workflow engine;
- separate services for very large modules;
- more advanced retry and compensation logic.

## 17.5 Advanced Knowledge

- automatic policy-version comparison;
- document quality checks;
- source-confidence scoring;
- knowledge-gap detection.

## 17.6 Advanced Review

- department-specific reviewers;
- multiple reviewer strategies;
- evaluation datasets;
- automated regression testing of prompts.

## 17.7 Analytics

- request completion time;
- department bottlenecks;
- human wait time;
- automation rate;
- capability-gap frequency;
- cost and LLM usage;
- simulated workflow performance.

## 17.8 Workflow Simulation

The platform may later simulate:

- human delays;
- supplier response time;
- physical delivery;
- approval latency;
- department workload.

## 17.9 Model Routing

A later version may assign different models to different departments or request categories when cost and quality justify it.
